# 🍽️ Danny Diner SQL Case Study
SQL analysis of Danny Dinner dataset to extract business insights on customers and sales


## 📋 Project Overview
This project is part of the Danny's Diner case study from the #8WeekSQLChallenge. I acted as a data analyst for a fictional Japanese restaurant to help them understand customer visiting patterns and spending habits.

## 🛠 Tools Used
- SQL (MySQL)

## 📊 Key Insights
- Total spending of each customer
- Most popular menu item
- First item purchased by each customer
- Customer visit frequency
- Top Spender:Identified which customers are driving the most revenue.
- Loyalty Impact: Analyzed how becoming a member changed customer spending frequency.
  
  -> **These insights allow the restaurant to personalize loyalty reward and optimize inventory for most popular dishes**

## 📂 Dataset
Contains:
• sales
• menu
• members


## 🛠️ Skills Demonstrated
- **Complex Joins:** Connecting sales, menu, and membership tables.
- **Window Functions:** Using `ROW_NUMBER()`,`RANK()` and `DENSE_RANK()` for customer behavior analysis.
- **CTEs:** Creating temporary result sets for multi-step calculations.
- **Conditional Logic:** Using `CASE WHEN` for points-based rewards systems.
